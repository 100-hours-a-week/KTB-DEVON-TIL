# 추천 조합 (포트폴리오 임팩트 우선)

## 추천 1순위 조합
- `_1 Cache Stampede`
- `_5 Offset drift -> Cursor 전환`
- `_6 DEL 폭탄 -> Versioning 무효화`
- `_3 직렬화 경량화` (추가 권장)

## 추천 이유
- 운영 환경에서 자주 발생하는 문제를 포함한다.
- `P99`, `DB read burst`, `hit ratio`, `Redis CPU`, `앱 CPU`까지 폭넓은 개선 지표를 확보할 수 있다.
- 단순 조회 최적화가 아니라 장애 예방/비용 절감/확장성까지 연결되는 스토리다.

## 발표 구성 예시
1. 장애 재현: 피크 구간에서 지표 붕괴 확인
2. 원인 분리: 데이터 구조/무효화/직렬화 각 축으로 분석
3. 순차 개선: cursor -> versioning -> stampede 방지 -> 직렬화
4. 정량 결과: 전/후 비교표 + 그래프

## 최소 제출 산출물
- 전/후 지표 표 1개
- Grafana 오버레이 이미지 1개
- `k6` 리포트 1개
- Redis slowlog 캡처 1개
