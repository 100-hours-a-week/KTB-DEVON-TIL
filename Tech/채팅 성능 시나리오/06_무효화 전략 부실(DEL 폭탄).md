# _6 무효화 전략 부실(DEL 폭탄)

## 문제 정의
수정/삭제 시 광범위 `DEL`을 동기 수행하면 무효화 시점에 Redis 지연 스파이크와 timeout이 발생한다.

## 재현 조건
- 대화방별 다수 키를 사용하는 구조
- 이벤트 발생 시 `DEL chat:{id}:*` 패턴 실행
- 피크 시간대 무효화 이벤트를 집중 발생

## 개선 전략
- versioned key(`v{version}`) 도입
- 이벤트 시 version 증가로 즉시 전환
- 구버전 키는 TTL 기반 lazy expiry

## 검증 지표
- 무효화 시점 `P99` spike
- timeout 비율
- slowlog 내 삭제 명령 빈도

## 포트폴리오 포인트
운영 장애 예방 관점에서 안정성을 높인 개선 사례로 설명하기 좋다.
