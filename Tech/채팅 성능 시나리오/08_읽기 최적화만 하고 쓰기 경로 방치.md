# _8 읽기 최적화만 하고 쓰기 경로 방치

## 문제 정의
읽기 경로만 최적화하고 write-through/write-behind 전략을 명확히 하지 않으면 지연과 정합성 이슈가 반복된다.

## 재현 조건
- 쓰기 후 읽기 지연 민감 구간 정의
- 장애/재시도 상황에서 중복 쓰기 유발
- 비동기 저장 실패 시 재처리 시나리오 점검

## 개선 전략
- 이벤트 기반 비동기 저장(Kafka/Redis Stream)
- idempotency key 기반 중복 방지
- 재처리 정책(DLQ, retry budget) 명시

## 검증 지표
- write latency
- 실패 재처리율
- 데이터 정합성 지표(중복/유실/역전)

## 포트폴리오 포인트
성능뿐 아니라 신뢰성과 확장성까지 고려한 아키텍처 설계 역량을 보여준다.
