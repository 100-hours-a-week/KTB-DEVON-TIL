# Pipeline vs MULTI/EXEC

## 정의
Pipeline은 여러 명령을 한 번에 전송해 RTT를 줄이는 최적화이고, MULTI/EXEC은 여러 명령을 트랜잭션처럼 묶어 원자성을 확보하는 기능입니다.
목적이 성능인지 원자성인지에 따라 선택이 달라집니다.

## 특징
- Pipeline: 네트워크 왕복 감소에 강점, 원자성 보장은 없음
- MULTI/EXEC: 순차 실행/원자성 보장, RTT 최적화와는 별개
- Lua 스크립트는 복합 연산을 원자적으로 처리할 때 유용합니다.

## 실제 사용 예시(사례)
- 랭킹 조회 다건 요청은 Pipeline으로 묶어 지연을 줄입니다.
- 재고 차감+검증은 Lua 또는 MULTI/EXEC로 원자 처리합니다.

## 보조 개념 정리
- RTT: 요청-응답 왕복 시간
- 원자성: 연산이 전부 성공하거나 전부 실패하는 성질
- Lua Script: Redis 내에서 원자적으로 실행되는 스크립트
