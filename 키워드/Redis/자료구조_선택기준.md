# Redis 자료구조 선택 기준

## 정의
Redis는 문자열(String) 외에도 List, Set, Sorted Set, Hash 등 다양한 자료구조를 제공하며, 문제 유형에 맞는 구조 선택이 성능을 좌우합니다.

## 특징
- String: 단순 키-값, 카운터, 캐시 본문 저장에 적합
- List: 순서 있는 메시지 큐/최근 N개 로그 저장에 유리
- Set: 중복 없는 집합 연산(교집합/합집합)에 적합
- ZSet: 점수 기반 랭킹/우선순위 큐에 적합
- Hash: 객체 필드 단위 저장/갱신에 효율적

## 실제 사용 예시(사례)
- 채팅 최신 100개 메시지는 List로 유지(`LPUSH`, `LTRIM`)
- 실시간 인기 검색어는 ZSet으로 점수 증가(`ZINCRBY`) 후 상위 조회

## 보조 개념 정리
- 시간복잡도: 명령어별 O(1), O(logN), O(N) 특성
- 메모리 오버헤드: 자료구조별 내부 인코딩 차이
- 키 설계: prefix와 도메인 단위 분리 전략
