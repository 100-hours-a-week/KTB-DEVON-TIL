# TTL과 만료 전략

## 정의
TTL(Time To Live)은 키의 수명을 지정해 일정 시간이 지나면 자동 삭제되도록 하는 정책입니다.
캐시 일관성과 메모리 운영의 핵심 제어 장치입니다.

## 특징
- TTL이 없으면 cold key 누적으로 메모리 압박이 커집니다.
- TTL이 너무 짧으면 miss 급증으로 원본 저장소 부하가 증가합니다.
- 만료 시점 동시 집중은 stampede를 유발할 수 있습니다.

## 실제 사용 예시(사례)
- 최신 캐시는 1~5분, 과거 페이지 캐시는 5~30분으로 구간별 TTL을 다르게 둡니다.
- TTL jitter를 적용해 동시 만료를 분산합니다.

## 보조 개념 정리
- Jitter: 만료 시간을 랜덤 분산하는 기법
- Passive/Active Expire: 접근 시 만료 확인과 백그라운드 만료 정리
- Eviction: 메모리 한계 도달 시 키 축출 정책
