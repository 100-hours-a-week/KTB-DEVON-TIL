# Redis 분산락(Redisson)

## 정의
Redis 분산락은 여러 인스턴스가 같은 자원에 동시 접근하지 못하도록 제어하는 잠금 메커니즘입니다.
Redisson은 분산락, 재진입락, 공정락 등을 제공하는 Java 클라이언트입니다.

## 특징
- 락 점유 시간(lease time)과 대기 시간(wait time) 설정이 중요합니다.
- 락 해제 누락 시 장애로 이어질 수 있어 finally 해제가 필수입니다.
- 락만으로 정합성이 완전 보장되지 않으므로 DB 제약과 함께 사용해야 안전합니다.

## 실제 사용 예시(사례)
- 동일 쿠폰에 대한 중복 발급을 방지하기 위해 키 단위 락을 획득 후 처리합니다.
- 배치 중복 실행 방지를 위해 작업 키를 기준으로 분산락을 사용합니다.

## 보조 개념 정리
- Reentrant Lock: 같은 스레드 재진입 허용 락
- Watchdog: 락 만료 시간을 자동 연장하는 보호 메커니즘
- Fencing Token: 분산락 stale owner 문제 완화 토큰 전략
