# Cache Stampede 대응

## 정의
Cache Stampede는 인기 키가 동시에 만료되어 다수 요청이 원본 저장소로 몰리는 현상입니다.
짧은 순간에 DB/백엔드 과부하를 유발할 수 있습니다.

## 특징
- 트래픽 피크와 TTL 동시 만료가 겹치면 급격히 악화됩니다.
- 단일 키 병목이 전체 시스템 지연으로 확산될 수 있습니다.
- 예방 전략은 캐시 정책 + 동시성 제어를 함께 적용해야 효과가 큽니다.

## 실제 사용 예시(사례)
- Single-flight 락으로 한 요청만 원본 조회를 수행하고 나머지는 대기합니다.
- Stale-While-Revalidate로 오래된 값 임시 제공 후 백그라운드 갱신합니다.

## 보조 개념 정리
- Soft TTL: 사용자 응답 TTL과 내부 재생성 TTL 분리
- Mutex Lock: 재생성 동시 수행 방지 잠금
- Warmup: 미리 자주 쓰는 키를 채워두는 전략
