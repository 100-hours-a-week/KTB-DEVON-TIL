# 단위 테스트 vs 통합 테스트

## 정의
단위 테스트는 작은 단위(메서드/클래스) 로직을 빠르게 검증하는 테스트이고, 통합 테스트는 여러 컴포넌트가 연결된 실제 동작을 검증하는 테스트입니다.
둘은 대체 관계가 아니라 상호 보완 관계입니다.

## 특징
- 단위 테스트: 빠르고 원인 파악이 쉬우며, 분기 검증에 강합니다.
- 통합 테스트: 설정/트랜잭션/쿼리/보안 같은 실제 연동 오류를 잡습니다.
- 단위 테스트만으로는 프레임워크 설정 문제를 놓칠 수 있습니다.
- 통합 테스트만으로는 느려서 피드백 루프가 무거워질 수 있습니다.

## 실제 사용 예시(사례)
- 서비스 정책 분기는 단위 테스트로 빠르게 검증합니다.
- API 엔드투엔드 흐름은 통합 테스트로 상태 코드와 DB 반영까지 검증합니다.

## 보조 개념 정리
- 테스트 피라미드: 단위 테스트를 넓게, 통합/E2E를 선택적으로 배치하는 전략
- 회귀 테스트: 수정 후 기존 기능이 깨지지 않았는지 확인하는 테스트
- 테스트 격리: 테스트 간 상태 영향을 차단하는 원칙
