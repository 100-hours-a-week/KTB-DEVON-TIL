# TDD

## 정의
TDD(Test-Driven Development)는 테스트를 먼저 작성하고, 테스트를 통과시키는 최소 구현을 만든 뒤 리팩터링하는 개발 방식입니다.
짧은 피드백 루프로 설계 품질을 끌어올리는 것이 목적입니다.

## 특징
- Red -> Green -> Refactor 사이클로 진행합니다.
- 요구사항을 테스트로 명확히 표현할 수 있습니다.
- 과도하게 세부 구현을 고정하면 리팩터링 저항이 생길 수 있습니다.

## 실제 사용 예시(사례)
- 할인 정책 요구사항을 테스트 케이스로 먼저 작성한 뒤 구현합니다.
- 실패 테스트를 기준으로 최소 구현 후 중복 제거 리팩터링을 진행합니다.

## 보조 개념 정리
- Red: 실패하는 테스트 작성 단계
- Green: 테스트 통과를 위한 최소 구현 단계
- Refactor: 동작 유지한 채 구조 개선 단계
