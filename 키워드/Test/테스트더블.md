# 테스트 더블(Test Double)

## 정의
테스트 더블은 테스트에서 실제 의존 객체를 대체하기 위해 사용하는 가짜 객체의 총칭입니다.
Mock, Stub, Fake, Spy, Dummy가 대표 유형입니다.

## 특징
- 외부 API/DB/메시지 브로커 같은 의존성을 분리해 테스트 속도를 높입니다.
- 검증 목적에 따라 적절한 더블 타입을 선택해야 합니다.
- 과도한 Mock 사용은 구현 결합도를 높여 리팩토링 저항을 키울 수 있습니다.

## 실제 사용 예시(사례)
- Stub으로 고정 응답을 반환해 서비스 분기를 검증합니다.
- Mock으로 특정 메서드 호출 여부와 횟수를 검증합니다.
- Fake 저장소(인메모리)로 리포지토리 동작을 간단히 대체합니다.

## 보조 개념 정리
- Stub: 정해진 데이터를 반환하는 대역
- Mock: 호출 상호작용을 검증하는 대역
- Fake: 단순 구현을 가진 대체 객체(예: 인메모리 저장소)
