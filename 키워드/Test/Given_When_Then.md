# Given-When-Then

## 정의
Given-When-Then은 테스트를 준비, 실행, 검증 단계로 명확히 분리해 가독성과 의도를 높이는 작성 패턴입니다.
테스트 실패 시 원인 파악을 빠르게 해줍니다.

## 특징
- Given: 입력 데이터/사전 상태를 준비합니다.
- When: 테스트 대상 동작을 실행합니다.
- Then: 기대 결과를 검증합니다.
- 단계가 명확하면 테스트 유지보수 비용이 줄어듭니다.

## 실제 사용 예시(사례)
- Given: 잔액 10,000원 계정 준비
- When: 3,000원 결제 실행
- Then: 잔액이 7,000원인지 검증

## 보조 개념 정리
- Arrange-Act-Assert: Given-When-Then과 유사한 구조
- 테스트 의도: 무엇을 보장하려는 테스트인지 한 문장으로 설명 가능해야 함
- 단일 책임 테스트: 한 테스트는 한 가지 행동만 검증
