# TCP vs UDP

## 정의
TCP는 연결 지향·신뢰성 중심 전송 프로토콜이고, UDP는 비연결·저지연 중심 전송 프로토콜입니다.
요구사항(정확성 vs 지연)에 따라 선택이 달라집니다.

## 특징
- TCP: 순서 보장, 재전송, 흐름 제어 제공
- UDP: 오버헤드가 작고 빠르지만 손실/순서 보장을 하지 않음
- 실시간 스트리밍/게임은 UDP, 일반 웹 트래픽은 TCP 기반이 일반적입니다.

## 실제 사용 예시(사례)
- HTTP/1.1, HTTP/2는 TCP 기반으로 동작합니다.
- 실시간 음성 통신(WebRTC 일부 경로)은 UDP 기반으로 지연을 최소화합니다.

## 보조 개념 정리
- 3-way handshake: TCP 연결 수립 절차
- 재전송: 패킷 손실 시 데이터 복구 메커니즘
- RTT: 왕복 지연 시간
