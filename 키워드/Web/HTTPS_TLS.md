# HTTPS와 TLS

## 정의
HTTPS는 HTTP 통신을 TLS로 암호화해 기밀성·무결성·서버 인증을 보장하는 프로토콜입니다.
웹 보안의 기본 전제로, 평문 전송 위험을 줄입니다.

## 특징
- 전송 데이터가 암호화되어 중간자 공격 위험을 낮춥니다.
- 서버 인증서를 통해 통신 상대의 신뢰성을 확인합니다.
- TLS 핸드셰이크 비용이 있지만 현대 환경에서는 대부분 최적화되어 있습니다.

## 실제 사용 예시(사례)
- 로그인/결제 API는 HTTPS 강제가 필수입니다.
- `HSTS` 설정으로 브라우저가 항상 HTTPS로 접속하게 유도합니다.

## 보조 개념 정리
- 핸드셰이크: 암호화 알고리즘/키 교환을 협상하는 초기 절차
- 대칭키/비대칭키: 암호화 키 사용 방식의 차이
- 인증서 체인: 루트 CA부터 서버 인증서까지 신뢰 연결 구조
