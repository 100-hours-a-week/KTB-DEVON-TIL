# DNS

## 정의
DNS(Domain Name System)는 사람이 읽기 쉬운 도메인 이름을 IP 주소로 변환해 주는 분산형 네이밍 시스템입니다.
"www.example.com" 같은 이름을 실제 통신 가능한 주소로 바꿔 네트워크 연결을 가능하게 합니다.

## 특징
- 계층 구조(루트 DNS -> TLD DNS -> 권한 DNS)로 조회가 진행됩니다.
- 한 번 조회한 결과는 TTL 동안 로컬/리졸버 캐시에 저장되어 응답 속도를 높입니다.
- A, AAAA, CNAME, MX, TXT 등 레코드 타입별로 역할이 다릅니다.
- 완전 분산 구조라 특정 서버 하나가 전체 DNS를 소유하지 않습니다.

## 실제 사용 예시(사례)
- 브라우저에서 `https://www.naver.com` 입력 시, 먼저 DNS 조회로 대상 서버 IP를 확인한 뒤 TCP/TLS 연결을 시도합니다.
- 쿠버네티스에서는 서비스 디스커버리를 위해 내부 DNS(`service-name.namespace.svc.cluster.local`)를 사용합니다.

## 보조 개념 정리
- TTL: DNS 응답 캐시 유지 시간
- 재귀 질의: 클라이언트 대신 리졸버가 최종 IP까지 조회하는 방식
- 권한 DNS: 특정 도메인에 대한 최종 정답 레코드를 가진 서버
