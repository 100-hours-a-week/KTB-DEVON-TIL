# 쿠키와 세션

## 정의
쿠키는 클라이언트(브라우저)에 저장되는 작은 데이터이고, 세션은 서버가 사용자 상태를 식별·관리하는 메커니즘입니다.
일반적으로 세션 ID를 쿠키로 전달해 서버의 세션 저장소와 연결합니다.

## 특징
- 쿠키는 요청마다 자동 전송되어 상태 유지에 사용됩니다.
- 세션 데이터는 서버에 있으므로 민감정보를 직접 노출하지 않기 쉽습니다.
- 세션 기반 인증은 서버 저장소 비용이 발생합니다.
- 쿠키는 `HttpOnly`, `Secure`, `SameSite` 설정으로 보안 수준을 높일 수 있습니다.

## 실제 사용 예시(사례)
- 로그인 성공 후 서버가 `SESSIONID`를 발급하고 쿠키로 내려줍니다.
- 이후 요청에서 브라우저가 쿠키를 자동 전송하면 서버가 세션을 조회해 인증 상태를 확인합니다.

## 보조 개념 정리
- Stateless: 서버가 요청 간 상태를 저장하지 않는 설계
- 세션 고정 공격: 공격자가 미리 아는 세션 ID를 사용하게 만드는 공격
- SameSite: 크로스 사이트 요청 시 쿠키 전송 정책
