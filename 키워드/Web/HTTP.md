# HTTP

## 정의
HTTP(HyperText Transfer Protocol)는 클라이언트와 서버가 요청/응답 방식으로 데이터를 주고받는 애플리케이션 계층 프로토콜입니다.
웹 브라우저가 서버에 리소스를 요청하고, 서버가 상태 코드와 본문을 응답하는 표준 통신 규약입니다.

## 특징
- Stateless 프로토콜이라 각 요청은 독립적으로 처리됩니다.
- 메서드(GET, POST, PUT, DELETE 등)로 의도를 표현합니다.
- 상태 코드를 통해 처리 결과(2xx 성공, 4xx 클라이언트 오류, 5xx 서버 오류)를 전달합니다.
- 헤더를 통해 캐시 정책, 인증 정보, 콘텐츠 타입 같은 메타데이터를 전달합니다.

## 실제 사용 예시(사례)
- REST API에서 `GET /users/{id}`로 사용자 조회, `POST /users`로 사용자 생성 요청을 처리합니다.
- CDN과 브라우저 캐시는 `Cache-Control`, `ETag`를 이용해 불필요한 네트워크 요청을 줄입니다.

## 보조 개념 정리
- URI: 리소스를 식별하는 문자열
- 멱등성: 같은 요청을 여러 번 보내도 결과가 동일한 성질(예: PUT)
- 콘텐츠 협상: `Accept` 헤더 기반으로 응답 포맷을 선택하는 방식
