# REST API

## 정의
REST API는 리소스를 URI로 식별하고 HTTP 메서드로 행위를 표현하는 아키텍처 스타일입니다.
일관된 리소스 모델과 상태 코드를 통해 클라이언트-서버 상호작용을 단순화합니다.

## 특징
- 리소스 중심 URL 설계(`/users/{id}`)를 사용합니다.
- 메서드 의미(GET/POST/PUT/DELETE)와 멱등성을 고려해야 합니다.
- 상태 코드를 표준화하면 클라이언트 처리 로직이 단순해집니다.

## 실제 사용 예시(사례)
- 사용자 조회: `GET /users/{id}`
- 사용자 생성: `POST /users`
- 사용자 수정: `PATCH /users/{id}`

## 보조 개념 정리
- HATEOAS: 응답에 다음 행위 링크를 포함하는 방식
- 멱등성: 같은 요청 반복 시 결과 일관성
- 표현(Representation): JSON/XML 같은 리소스 표현 포맷
