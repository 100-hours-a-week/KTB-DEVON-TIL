# 역색인(Inverted Index)

## 정의
역색인은 "단어 -> 문서 목록" 형태로 저장하는 색인 구조로, 검색어에 해당하는 문서를 빠르게 찾기 위한 핵심 자료구조입니다.
Elasticsearch 검색 성능의 기반입니다.

## 특징
- 전문 검색에서 포함 검색을 빠르게 수행할 수 있습니다.
- 토큰 단위로 인덱싱하므로 분석기 설정이 결과에 큰 영향을 줍니다.
- 쓰기 시 색인 비용이 들지만 조회 성능을 크게 개선합니다.

## 실제 사용 예시(사례)
- "spring" 검색 시 역색인에서 해당 토큰을 가진 문서 ID 목록을 즉시 조회합니다.
- 여러 토큰 쿼리는 교집합/합집합 계산으로 결과를 좁혀갑니다.

## 보조 개념 정리
- Posting List: 특정 토큰이 등장한 문서 목록
- Tokenization: 텍스트를 검색 가능한 토큰으로 분해하는 과정
- Scoring: 문서 관련도를 수치화하는 과정
