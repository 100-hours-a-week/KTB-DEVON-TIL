# Mapping

## 정의
Mapping은 문서 필드의 타입과 인덱싱 방식을 정의하는 Elasticsearch 스키마입니다.
검색 품질과 저장 비용, 집계 가능 여부를 좌우합니다.

## 특징
- 동적 매핑은 빠르지만 예상치 못한 필드 타입 오염 위험이 있습니다.
- `text`와 `keyword` 타입 목적이 다르므로 필드 의도에 맞게 선택해야 합니다.
- 잘못된 매핑은 재색인 없이는 수정이 어려운 경우가 많습니다.

## 실제 사용 예시(사례)
- 상품명은 `text`(검색), 상품코드는 `keyword`(정확 일치/집계)로 분리합니다.
- 날짜 필드는 명시 타입으로 정의해 정렬/범위 쿼리 정확성을 확보합니다.

## 보조 개념 정리
- Dynamic Mapping: 문서 입력 시 자동 필드 타입 추론
- Multi-field: 하나의 원본 필드를 text+keyword로 동시 인덱싱
- Reindex: 새로운 매핑으로 데이터를 재적재하는 작업
