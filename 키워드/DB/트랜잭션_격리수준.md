# 트랜잭션 격리수준

## 정의
트랜잭션 격리수준은 동시에 실행되는 트랜잭션 간에 데이터 변경이 서로 어떻게 보일지 결정하는 규칙입니다.
격리수준이 높을수록 일관성은 좋아지지만, 동시성 처리량은 감소할 수 있습니다.

## 특징
- `READ UNCOMMITTED`: 커밋 전 데이터도 읽을 수 있어 Dirty Read 위험이 큽니다.
- `READ COMMITTED`: 커밋된 데이터만 읽어 Dirty Read를 방지합니다.
- `REPEATABLE READ`: 같은 트랜잭션 내 동일 조회 결과를 보장해 Non-repeatable Read를 줄입니다.
- `SERIALIZABLE`: 가장 강한 격리로 팬텀 리드까지 방지하지만 락 경합이 큽니다.

| 격리수준 | Dirty Read | Non-repeatable Read | Phantom Read | 동시성 |
|---|---|---|---|---|
| READ UNCOMMITTED | 가능 | 가능 | 가능 | 높음 |
| READ COMMITTED | 방지 | 가능 | 가능 | 중간 |
| REPEATABLE READ | 방지 | 방지 | DB별 상이 | 중간~낮음 |
| SERIALIZABLE | 방지 | 방지 | 방지 | 낮음 |

## 실제 사용 예시(사례)
- 일반 CRUD 서비스는 `READ COMMITTED`를 기본으로 두고 시작합니다.
- 재고/정산 같이 정합성이 중요한 구간은 `REPEATABLE READ` 이상 또는 명시적 락을 검토합니다.

## 보조 개념 정리
- Dirty Read: 다른 트랜잭션이 아직 커밋하지 않은 변경을 읽는 현상
- Non-repeatable Read: 같은 조건 재조회 시 값이 바뀌는 현상
- Phantom Read: 같은 조건 범위 조회에서 행 개수가 달라지는 현상
