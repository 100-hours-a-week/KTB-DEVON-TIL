# 실행 계획(EXPLAIN)

## 정의
실행 계획은 SQL을 DB가 어떤 순서와 방식으로 실행할지 보여주는 계획 정보입니다.
성능 튜닝의 출발점은 실행 계획 확인입니다.

## 특징
- 인덱스 사용 여부, 조인 순서, 스캔 방식 등을 확인할 수 있습니다.
- 같은 SQL도 통계 정보/파라미터에 따라 계획이 달라질 수 있습니다.
- "느리다"는 감이 아니라 계획 기반으로 병목 원인을 좁혀야 합니다.

## 실제 사용 예시(사례)
- Full Scan이 보이면 인덱스 후보와 조건절 선택도를 함께 점검합니다.
- 정렬 비용이 크면 `ORDER BY` 인덱스 정합성을 검토합니다.

## 보조 개념 정리
- Cardinality: 단계별 예상 처리 행 수
- Access Type: index, range, ref, all 등 접근 방식
- 비용 기반 최적화기: 통계 기반으로 최적 실행 경로를 선택하는 엔진
