# 정규화

## 정의
정규화는 데이터 중복을 줄이고 이상 현상(삽입/수정/삭제 이상)을 방지하기 위해 테이블 구조를 체계적으로 분해하는 설계 기법입니다.
무결성과 유지보수성을 높이는 것이 주 목적입니다.

## 특징
- 1NF, 2NF, 3NF를 거치며 중복과 함수 종속 문제를 줄입니다.
- 데이터 변경 시 일관성 유지가 쉬워집니다.
- 조인이 늘어 조회 비용이 증가할 수 있어 상황에 따라 반정규화가 필요합니다.

## 실제 사용 예시(사례)
- 주문 테이블에 사용자 이름을 중복 저장하지 않고 `user_id`로 분리합니다.
- 카테고리 정보를 별도 테이블로 분리해 수정 시 한 곳만 변경합니다.

## 보조 개념 정리
- 함수 종속: 특정 컬럼이 다른 컬럼 값을 결정하는 관계
- 이상 현상: 중복 구조에서 발생하는 데이터 불일치 문제
- 반정규화: 조회 성능을 위해 일부 중복을 의도적으로 허용하는 설계
