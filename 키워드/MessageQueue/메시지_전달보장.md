# 메시지 전달 보장

## 정의
메시지 전달 보장은 메시지가 몇 번 전달/처리되는지를 정의하는 약속입니다.
대표적으로 at-most-once, at-least-once, exactly-once가 있습니다.

## 특징
- at-most-once: 중복은 없지만 유실 가능성이 큼
- at-least-once: 유실은 줄지만 중복 처리를 고려해야 함
- exactly-once: 가장 이상적이지만 비용과 복잡도가 높음

## 실제 사용 예시(사례)
- 결제 이벤트는 at-least-once + idempotency key 조합으로 안정성을 확보합니다.
- 실시간 로그 스트림은 일부 유실 허용 시 at-most-once를 선택하기도 합니다.

## 보조 개념 정리
- Idempotency Key: 중복 처리 방지 식별자
- Retry Backoff: 재시도 간격을 점진적으로 늘리는 정책
- Deduplication: 중복 메시지 제거 처리
