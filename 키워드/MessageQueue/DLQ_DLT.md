# DLQ/DLT

## 정의
DLQ(Dead Letter Queue) 또는 DLT(Dead Letter Topic)는 처리 실패 메시지를 본 흐름에서 분리해 저장하는 공간입니다.
무한 재시도와 장애 확산을 막는 안전 장치입니다.

## 특징
- 본 큐/토픽의 처리 지연 전파를 줄입니다.
- 실패 원인 분석과 재처리 전략 수립이 쉬워집니다.
- 실패 메시지 스키마/메타데이터를 함께 저장해야 운영성이 좋아집니다.

## 실제 사용 예시(사례)
- 역직렬화 실패, 비즈니스 검증 실패 메시지를 DLT로 보내 별도 리커버리 워커에서 재처리합니다.
- 반복 실패 임계치 초과 시 알림을 발송합니다.

## 보조 개념 정리
- Poison Message: 어떤 소비자에서도 반복 실패하는 메시지
- Retry Topic: 재시도 간격별 분리 토픽
- Observability: 실패 추적을 위한 메트릭/로그/트레이싱 체계
