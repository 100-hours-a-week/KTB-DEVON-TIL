# Pub/Sub vs Queue

## 정의
Queue는 작업을 한 소비자가 처리하는 모델이고, Pub/Sub는 발행된 메시지를 여러 구독자가 각각 수신하는 모델입니다.
메시지 전달 목적(분배 vs 전파)에 따라 선택합니다.

## 특징
- Queue: 경쟁 소비(Competing Consumer)로 작업 분산에 적합
- Pub/Sub: 동일 이벤트를 다수 시스템에 전달할 때 적합
- 재시도/순서/중복 처리 정책은 구현체별로 다릅니다.

## 실제 사용 예시(사례)
- 이미지 리사이징 작업은 Queue로 워커가 분담 처리합니다.
- 주문 완료 이벤트는 Pub/Sub로 알림/정산/추천 시스템이 각각 소비합니다.

## 보조 개념 정리
- Fan-out: 하나의 이벤트를 여러 소비자로 확산하는 패턴
- Consumer Group: 그룹 단위로 메시지 처리 책임을 분담하는 구조
- Dead Letter Queue: 실패 메시지 격리 큐
