# Network RTT 왕복 과다

## 정의
RTT 왕복 과다는 한 요청 처리에 필요한 네트워크 호출 횟수가 많아져 누적 지연이 커지는 현상입니다.
캐시/DB/외부 API를 직렬 호출할 때 자주 발생합니다.

## 특징
- 개별 호출이 빨라도 왕복 횟수가 많으면 총 지연이 크게 증가합니다.
- 피크 트래픽에서 커넥션 경합과 큐잉 지연이 추가됩니다.
- 배치 처리/파이프라인/병렬화로 완화 가능합니다.

## 실제 사용 예시(사례)
- 채팅 조회에서 Redis 6회 + DB 1회 순차 호출로 P99 지연 급증
- Redis pipeline + 조회 경로 통합으로 RTT 절감

## 보조 개념 정리
- N+1 Remote Call: 루프 내부 원격 호출 반복 패턴
- Batching: 여러 요청을 묶어 호출 횟수를 줄이는 기법
- Queueing Delay: 처리 대기로 발생하는 지연
