# Hot Key 편중

## 정의
Hot Key 편중은 일부 키에 요청이 과도하게 집중되어 캐시/스토어의 특정 샤드나 노드가 병목이 되는 현상입니다.
전체 평균은 정상이어도 tail latency를 크게 악화시킵니다.

## 특징
- 인기 대화방, 공지 채널, 랭킹 키에서 자주 발생합니다.
- 단일 노드 CPU/네트워크 편중으로 장애 전조가 나타납니다.
- 키 분산/복제/로컬 캐시 전략으로 완화할 수 있습니다.

## 실제 사용 예시(사례)
- 한 채팅방 조회가 트래픽 40%를 차지해 특정 Redis shard가 포화됨
- 키 샤딩 접미사 + short-lived local cache로 부하 분산

## 보조 개념 정리
- Tail Latency: 상위 지연 구간(P95/P99)
- Key Sharding: 하나의 논리 키를 여러 물리 키로 분산
- Replica Read: 읽기 트래픽을 복제본으로 분산하는 전략
