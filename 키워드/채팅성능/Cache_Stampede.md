# Cache Stampede

## 정의
캐시 스탬피드는 동일 키가 만료되는 순간 다수 요청이 원본 저장소를 동시에 조회해 과부하를 만드는 현상입니다.
채팅 히스토리처럼 핫키가 있는 시스템에서 빈번히 발생합니다.

## 특징
- TTL 동시 만료와 피크 트래픽이 결합될 때 급격히 악화됩니다.
- DB 읽기 급증, 응답 지연 상승, 타임아웃 확산이 연쇄적으로 발생합니다.
- 방어 전략은 락/재검증/만료 분산을 함께 적용해야 안정적입니다.

## 실제 사용 예시(사례)
- 대화방 최신 20개 조회 키가 동시에 만료되어 Redis miss 폭증
- single-flight + stale-while-revalidate로 완화

## 보조 개념 정리
- Hot Key: 특정 시점 접근이 집중되는 키
- Soft TTL: 논리 만료와 물리 만료를 분리한 정책
- Herd Effect: 다수 요청이 같은 리소스로 몰리는 현상
