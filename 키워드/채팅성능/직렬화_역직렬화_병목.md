# 직렬화/역직렬화 병목

## 정의
직렬화/역직렬화 병목은 객체를 문자열/바이너리로 변환하는 비용이 요청 처리 시간에서 큰 비중을 차지하는 현상입니다.
고빈도 API에서 CPU 사용량을 급격히 높일 수 있습니다.

## 특징
- payload 크기가 커질수록 비용이 선형 이상 증가할 수 있습니다.
- 불필요한 필드 포함/깊은 객체 구조가 병목을 악화합니다.
- 포맷(JSON vs 바이너리), 라이브러리, DTO 설계가 성능 차이를 만듭니다.

## 실제 사용 예시(사례)
- 채팅 목록 응답에 사용하지 않는 메타데이터까지 포함돼 역직렬화 비용이 증가
- 경량 DTO 분리와 필드 축소로 CPU 사용량 완화

## 보조 개념 정리
- Payload: 네트워크로 실제 전달되는 데이터 본문
- DTO Projection: 필요한 필드만 담는 전송 객체 전략
- Object Allocation: 객체 생성/해제 비용으로 인한 GC 부담
