# L4 vs L7 Load Balancer

## 정의
L4 로드밸런서는 TCP/UDP 계층 정보로 트래픽을 분산하고, L7 로드밸런서는 HTTP 헤더/경로 등 애플리케이션 정보를 기준으로 분산합니다.

## 특징
- L4: 빠르고 단순하며 프로토콜 독립성이 높음
- L7: 라우팅 유연성, 인증/헤더 기반 정책 적용 가능
- 요구 기능이 단순하면 L4, 세밀한 라우팅이 필요하면 L7이 유리합니다.

## 실제 사용 예시(사례)
- 게임/저지연 TCP 서비스는 L4를 사용합니다.
- 웹 API 게이트웨이는 경로 기반 라우팅을 위해 L7을 사용합니다.

## 보조 개념 정리
- Health Check: 백엔드 서버 상태 점검
- Sticky Session: 동일 사용자를 같은 서버로 고정하는 정책
- TLS Termination: LB에서 TLS를 종료하는 방식
