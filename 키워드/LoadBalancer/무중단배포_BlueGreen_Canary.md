# Blue/Green vs Canary 배포

## 정의
Blue/Green은 신규 환경을 별도로 준비한 뒤 트래픽을 전환하는 배포 방식이고, Canary는 일부 트래픽부터 점진적으로 신규 버전을 노출하는 방식입니다.

## 특징
- Blue/Green: 롤백이 빠르지만 인프라 비용이 큼
- Canary: 위험을 분산하지만 모니터링/제어 복잡도가 높음
- 두 방식 모두 LB 라우팅 제어와 지표 기반 판정이 핵심입니다.

## 실제 사용 예시(사례)
- 핵심 결제 서비스는 5% 카나리 -> 25% -> 100% 단계로 확장합니다.
- 대형 릴리즈는 Blue/Green으로 전환 후 문제 시 즉시 이전 환경으로 롤백합니다.

## 보조 개념 정리
- Progressive Delivery: 점진 노출 기반 배포 전략
- Rollback: 문제 발생 시 이전 버전으로 복귀
- Error Budget: 허용 가능한 장애/오류 범위
