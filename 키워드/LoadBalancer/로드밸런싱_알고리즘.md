# 로드밸런싱 알고리즘

## 정의
로드밸런싱 알고리즘은 요청을 어떤 백엔드 인스턴스에 보낼지 결정하는 규칙입니다.
트래픽 패턴과 서버 특성에 맞는 알고리즘 선택이 중요합니다.

## 특징
- Round Robin: 단순 순환 분배, 구현이 쉽습니다.
- Least Connections: 현재 연결 수가 적은 서버 우선
- Weighted 방식: 서버 성능 차이를 반영해 분배 비율 조정
- IP Hash: 같은 클라이언트를 동일 서버로 유도

## 실제 사용 예시(사례)
- 성능 차이가 큰 혼합 노드 환경은 Weighted Round Robin을 사용합니다.
- 세션 의존 서비스는 IP Hash 또는 세션 스티키 전략을 적용합니다.

## 보조 개념 정리
- Throughput: 단위 시간 처리량
- Tail Latency: 상위 백분위 지연 시간(P95/P99)
- Slow Start: 신규 인스턴스에 트래픽을 점진 분배하는 기법
