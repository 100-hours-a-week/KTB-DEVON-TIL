# JIT 컴파일러

## 정의
JIT(Just-In-Time) 컴파일러는 실행 중 자주 호출되는 바이트코드를 네이티브 코드로 변환해 성능을 높이는 JVM 최적화 기능입니다.
처음에는 해석 실행하고, 충분히 뜨거워진 코드부터 컴파일합니다.

## 특징
- 핫스팟 코드 중심 최적화로 반복 경로 성능이 좋아집니다.
- 인라이닝, 탈출 분석, 루프 최적화 등 다양한 기법을 적용합니다.
- 워밍업 전후 성능이 달라질 수 있어 벤치마크 설계에 주의가 필요합니다.

## 실제 사용 예시(사례)
- 장시간 실행 서비스는 JIT 최적화 이후 처리량이 안정적으로 증가합니다.
- 성능 테스트 시 초기 워밍업 구간을 제외하고 측정해야 왜곡을 줄일 수 있습니다.

## 보조 개념 정리
- HotSpot: 자주 실행되는 코드 경로
- 인라이닝: 메서드 호출을 본문으로 치환해 오버헤드를 줄이는 최적화
- 워밍업: JIT 최적화가 진행되는 초기 실행 구간
