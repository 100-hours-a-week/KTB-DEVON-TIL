# JDK와 JRE

## 정의
JRE(Java Runtime Environment)는 자바 프로그램 실행 환경이고, JDK(Java Development Kit)는 JRE에 개발 도구(`javac`, `javadoc` 등)를 포함한 개발용 패키지입니다.
즉, 실행만 하면 JRE로 충분하고, 개발/컴파일까지 하려면 JDK가 필요합니다.

## 특징
- JDK는 JRE를 포함하므로 개발 PC에는 보통 JDK를 설치합니다.
- 최신 자바 배포판에서는 JRE를 별도로 제공하지 않거나, JDK 중심 배포가 일반적입니다.
- 빌드/테스트 파이프라인은 JDK 버전(예: 17, 21)에 강하게 의존합니다.

## 실제 사용 예시(사례)
- CI 서버에서 Gradle 빌드를 돌릴 때 JDK 17을 명시해 컴파일 호환성을 보장합니다.
- 서버 런타임 이미지는 경량화를 위해 JDK 전체 대신 jlink 기반 커스텀 런타임을 구성하기도 합니다.

## 보조 개념 정리
- `javac`: 자바 소스를 바이트코드로 컴파일하는 도구
- LTS: 장기 지원 버전(예: Java 17, 21)
- jlink: 필요한 모듈만 묶어 맞춤 런타임 이미지를 만드는 도구
