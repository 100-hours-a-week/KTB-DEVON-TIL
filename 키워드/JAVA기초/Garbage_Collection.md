# Garbage Collection

## 정의
가비지 컬렉션은 더 이상 참조되지 않는 객체를 자동으로 회수해 힙 메모리를 관리하는 JVM 기능입니다.
개발자가 직접 메모리 해제를 하지 않아도 되지만, GC 동작은 성능에 직접 영향을 줍니다.

## 특징
- 도달 가능성(Reachability) 기준으로 생존 객체를 판단합니다.
- Young/Old 영역을 분리해 세대별로 수집 전략을 다르게 가져갑니다.
- GC 빈도/정지 시간은 애플리케이션 지연 시간에 영향을 줍니다.

## 실제 사용 예시(사례)
- API 서버에서 GC pause가 길어지면 P99 응답 시간이 급증할 수 있어 GC 로그 분석이 필요합니다.
- G1GC, ZGC 같은 수집기를 서비스 특성(처리량/지연)에 맞춰 선택합니다.

## 보조 개념 정리
- STW(Stop-The-World): GC 중 애플리케이션 스레드가 멈추는 구간
- Minor/Full GC: Young 영역 중심 수집과 전체 영역 수집
- 객체 승격: Young에서 살아남은 객체가 Old로 이동하는 과정
