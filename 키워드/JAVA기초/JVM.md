# JVM

## 정의
JVM(Java Virtual Machine)은 자바 바이트코드를 실행하는 가상 머신입니다.
운영체제마다 JVM 구현체가 바이트코드를 해석/컴파일해 실행하므로, 같은 자바 프로그램을 여러 환경에서 실행할 수 있습니다.

## 특징
- 클래스 로딩, 바이트코드 검증, 실행 엔진, 메모리(Runtime Data Area)로 구성됩니다.
- JIT 컴파일러가 자주 실행되는 코드를 네이티브 코드로 최적화합니다.
- 가비지 컬렉션으로 힙 메모리를 자동 관리합니다.
- 플랫폼 독립성을 제공하지만, 성능은 JVM 옵션/GC 정책에 영향을 받습니다.

## 실제 사용 예시(사례)
- Spring Boot 애플리케이션은 `jar`로 패키징된 뒤 JVM 위에서 실행됩니다.
- 운영 환경에서 `-Xms`, `-Xmx`, `-XX:+UseG1GC` 같은 옵션으로 메모리/GC 특성을 조정합니다.

## 보조 개념 정리
- 바이트코드: JVM이 실행하는 중간 코드(.class)
- JIT: 실행 중 자주 쓰이는 코드를 기계어로 컴파일해 성능을 높이는 기술
- 힙/스택: 객체 저장 영역과 메서드 호출 정보를 저장하는 메모리 영역
