# IoC와 DI

## 정의
IoC(Inversion of Control)는 객체 생성/생명주기 제어권을 프레임워크가 갖는 개념이고, DI(Dependency Injection)는 필요한 의존성을 외부에서 주입받는 방식입니다.
Spring은 IoC 컨테이너를 통해 DI를 실현합니다.

## 특징
- 객체 간 결합도를 낮춰 테스트와 교체가 쉬워집니다.
- 생성자 주입을 기본으로 하면 불변성과 필수 의존성 보장이 좋아집니다.
- 설정 기반으로 구현체를 바꿔 끼울 수 있어 확장에 유리합니다.

## 실제 사용 예시(사례)
- `OrderService`가 `PaymentClient` 인터페이스를 생성자 주입받아 사용합니다.
- 테스트에서는 Mock 구현체를 주입해 외부 API 호출 없이 검증합니다.

## 보조 개념 정리
- Bean: Spring 컨테이너가 관리하는 객체
- 의존성 주입 방식: 생성자 주입, 수정자 주입, 필드 주입
- 컨테이너: 객체 생성/주입/초기화/소멸을 관리하는 런타임
