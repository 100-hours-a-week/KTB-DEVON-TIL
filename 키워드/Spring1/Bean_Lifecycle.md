# Bean 생명주기

## 정의
Bean 생명주기는 Spring 컨테이너가 Bean을 생성하고 의존성을 주입한 뒤 초기화하고, 종료 시 소멸시키는 전체 과정입니다.
초기화/정리 훅을 통해 리소스 준비와 해제를 제어할 수 있습니다.

## 특징
- 생성 -> 의존성 주입 -> 초기화 콜백 -> 사용 -> 소멸 콜백 순서로 진행됩니다.
- `@PostConstruct`, `@PreDestroy`로 초기화/정리 로직을 선언할 수 있습니다.
- 싱글톤은 컨테이너 종료 시점에 소멸되고, 프로토타입은 소멸 관리가 제한적입니다.

## 실제 사용 예시(사례)
- 외부 클라이언트 연결 풀을 초기화 시점에 준비하고 종료 시 안전하게 닫습니다.
- 캐시 워밍업 로직을 `@PostConstruct`에 배치해 시작 성능을 안정화합니다.

## 보조 개념 정리
- InitializingBean/DisposableBean: 생명주기 콜백 인터페이스
- BeanPostProcessor: Bean 생성 후처리 확장 포인트
- Scope별 생명주기: Bean 스코프에 따라 생성/소멸 시점이 달라짐
