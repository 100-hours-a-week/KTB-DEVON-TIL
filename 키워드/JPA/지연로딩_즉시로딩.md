# 지연 로딩 vs 즉시 로딩

## 정의
지연 로딩(LAZY)은 연관 엔티티를 실제 접근 시 조회하는 전략이고, 즉시 로딩(EAGER)은 엔티티 조회 시점에 함께 로딩하는 전략입니다.
기본 선택은 보통 LAZY이며, 필요한 쿼리에서만 명시적으로 fetch 전략을 조정합니다.

## 특징
- LAZY는 불필요한 조회를 줄여 기본 성능에 유리합니다.
- EAGER는 단순해 보이지만 예기치 않은 조인/추가 쿼리를 유발할 수 있습니다.
- 직렬화 시점에 LAZY 초기화 예외가 발생할 수 있어 계층 분리가 중요합니다.
- 실무에서는 Fetch Join/DTO 조회로 필요한 데이터만 명시적으로 가져옵니다.

## 실제 사용 예시(사례)
- 목록 API는 DTO 프로젝션으로 필요한 컬럼만 조회합니다.
- 상세 API는 Fetch Join으로 연관 데이터를 한 번에 가져와 N+1을 방지합니다.

## 보조 개념 정리
- 프록시: 실제 엔티티 로딩 전 대리 객체
- LazyInitializationException: 영속성 컨텍스트 종료 후 지연 로딩 접근 시 발생
- Fetch Join: 연관 엔티티를 조인해 한 번에 조회하는 방법
