# 영속성 컨텍스트

## 정의
영속성 컨텍스트는 엔티티를 관리하는 1차 캐시이자 변경 감지 단위입니다.
JPA는 같은 트랜잭션 안에서 영속성 컨텍스트를 통해 객체 동일성과 쓰기 지연을 보장합니다.

## 특징
- 1차 캐시: 같은 ID 엔티티를 재조회하면 DB 대신 캐시에서 반환합니다.
- 변경 감지(Dirty Checking): 트랜잭션 종료 시 변경된 필드를 자동 반영합니다.
- 쓰기 지연: SQL을 바로 보내지 않고 flush 시점에 모아 실행합니다.
- 준영속 상태가 되면 변경 감지가 동작하지 않습니다.

## 실제 사용 예시(사례)
- 서비스 트랜잭션에서 `findById` 후 필드만 변경해도 명시적 `save` 없이 업데이트가 반영됩니다.
- 대량 처리에서는 주기적으로 `flush`/`clear`를 호출해 메모리 사용량을 제어합니다.

## 보조 개념 정리
- Entity Life Cycle: 비영속, 영속, 준영속, 삭제 상태
- Flush: 영속성 컨텍스트 변경 내용을 DB로 동기화하는 시점
- Detach: 엔티티를 컨텍스트 관리 대상에서 분리하는 동작
