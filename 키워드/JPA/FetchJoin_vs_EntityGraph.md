# Fetch Join vs EntityGraph

## 정의
Fetch Join은 JPQL에서 연관 엔티티를 조인해 즉시 로딩하는 방법이고, EntityGraph는 조회 시점에 로딩 그래프를 선언적으로 지정하는 방법입니다.
둘 다 N+1 완화에 사용되지만 표현 방식과 재사용성이 다릅니다.

## 특징
- Fetch Join: 쿼리에서 명시적 제어가 쉬움, 페이징과 조합 시 제약이 있을 수 있음
- EntityGraph: 리포지토리 메서드에 선언해 재사용하기 좋음
- 상황에 따라 DTO 조회가 더 단순하고 효율적일 수 있습니다.

## 실제 사용 예시(사례)
- 상세 조회는 Fetch Join으로 필요한 연관을 한 번에 가져옵니다.
- 여러 조회 메서드에서 같은 로딩 정책이 필요하면 EntityGraph를 선언해 중복을 줄입니다.

## 보조 개념 정리
- N+1: 연관 조회 시 반복 쿼리 폭증 문제
- 즉시 로딩 제어: 필요한 쿼리에서만 연관 로딩을 강제하는 전략
- DTO 프로젝션: 엔티티 대신 필요한 필드만 조회하는 방식
