# Cascade와 OrphanRemoval

## 정의
Cascade는 부모 엔티티의 영속성 작업을 자식 엔티티에 전파하는 옵션이고, OrphanRemoval은 부모와 연관이 끊긴 자식을 자동 삭제하는 옵션입니다.
연관관계 생명주기 관리를 단순화하지만 오남용 시 데이터 손실 위험이 있습니다.

## 특징
- `CascadeType.PERSIST`로 부모 저장 시 자식도 함께 저장할 수 있습니다.
- `CascadeType.REMOVE`는 부모 삭제 시 자식 삭제를 전파합니다.
- `orphanRemoval=true`는 컬렉션에서 제거된 자식을 삭제 대상으로 봅니다.
- 전파 범위가 넓으면 의도치 않은 delete/update가 발생할 수 있습니다.

## 실제 사용 예시(사례)
- 주문-주문상품 관계에서 주문 저장 시 주문상품도 함께 저장합니다.
- 주문상품 목록에서 항목 제거 시 orphanRemoval로 불필요한 데이터가 정리됩니다.

## 보조 개념 정리
- Aggregate: 함께 일관성 경계를 갖는 객체 묶음
- 영속성 전이: 부모 작업을 자식으로 전달하는 메커니즘
- 연관관계 주인: 외래키를 실제로 관리하는 쪽
