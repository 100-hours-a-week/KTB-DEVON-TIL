# 토픽, 파티션, 오프셋

## 정의
토픽은 메시지를 분류하는 논리 채널이고, 파티션은 토픽을 나눈 물리 단위이며, 오프셋은 파티션 내 메시지 순번입니다.
Kafka는 이 3요소로 대용량 스트림을 병렬 처리합니다.

## 특징
- 파티션 단위로 순서가 보장됩니다.
- 오프셋 기반으로 재처리/이어읽기가 가능합니다.
- 파티션 수가 처리량 상한에 직접 영향을 줍니다.

## 실제 사용 예시(사례)
- `chat-message` 토픽을 파티션 12개로 구성해 소비자 병렬 처리량을 늘립니다.
- 장애 복구 후 마지막 커밋 오프셋부터 재소비해 유실을 줄입니다.

## 보조 개념 정리
- Key: 파티션 라우팅 기준 값
- Retention: 메시지 보관 기간/용량 정책
- Log Append: 메시지를 끝에 추가하는 저장 모델
