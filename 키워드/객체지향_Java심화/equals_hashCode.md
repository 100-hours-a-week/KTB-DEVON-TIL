# equals와 hashCode

## 정의
equals는 객체의 논리적 동등성을 비교하고, hashCode는 해시 기반 컬렉션에서 객체 위치를 결정하는 값입니다.
두 메서드는 함께 재정의해야 계약을 지킬 수 있습니다.

## 특징
- equals가 true면 hashCode도 같아야 합니다.
- hashCode만 같다고 equals가 true일 필요는 없습니다.
- 불변 필드를 기준으로 구현해야 컬렉션 동작이 안정적입니다.

## 실제 사용 예시(사례)
- `HashSet`에서 중복 제거가 기대와 다르게 동작하면 equals/hashCode 구현 누락을 먼저 점검합니다.
- JPA 엔티티는 영속성 식별자 시점 차이 때문에 구현 전략을 신중히 선택해야 합니다.

## 보조 개념 정리
- 동등성: 값 기준으로 같은지 판단하는 개념
- 동일성: 메모리 주소(참조)가 같은지 판단하는 개념
- 해시 충돌: 다른 객체가 같은 해시 값을 갖는 상황
