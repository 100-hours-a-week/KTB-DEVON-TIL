# SOLID

## 정의
SOLID는 객체지향 설계를 유연하고 유지보수 가능하게 만들기 위한 5가지 원칙 집합입니다.
변경에 강한 구조를 만들기 위해 "책임 분리"와 "추상화 의존"을 강조합니다.

## 특징
- SRP: 하나의 클래스는 하나의 변경 이유만 가져야 합니다.
- OCP: 기존 코드를 크게 수정하지 않고 확장 가능해야 합니다.
- LSP: 하위 타입은 상위 타입 계약을 깨면 안 됩니다.
- ISP: 큰 인터페이스보다 역할별 작은 인터페이스가 유리합니다.
- DIP: 구체 클래스보다 추상화에 의존해야 결합도를 낮출 수 있습니다.

## 실제 사용 예시(사례)
- 결제 모듈에서 `PaymentProcessor` 인터페이스를 두고 카드/계좌 구현체를 분리합니다.
- 신규 결제수단 추가 시 기존 서비스 코드를 최소 수정으로 확장합니다.

## 보조 개념 정리
- 응집도: 모듈 내부 요소가 관련된 정도
- 결합도: 모듈 간 의존 강도
- 의존성 역전: 상위 정책이 하위 구현에 직접 의존하지 않는 구조
