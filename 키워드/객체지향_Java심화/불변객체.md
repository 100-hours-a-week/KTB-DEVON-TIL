# 불변 객체

## 정의
불변 객체는 생성 이후 내부 상태가 변경되지 않는 객체입니다.
상태 변경이 필요하면 기존 객체를 수정하지 않고 새 객체를 생성합니다.

## 특징
- 멀티스레드 환경에서 동기화 부담을 줄입니다.
- 상태 추적이 쉬워 버그 원인 분석이 수월합니다.
- 객체 생성 비용이 증가할 수 있어 큰 데이터 구조에서는 전략적 사용이 필요합니다.

## 실제 사용 예시(사례)
- `String`, `LocalDateTime` 같은 표준 타입은 대표적인 불변 객체입니다.
- 도메인 값 객체(Money, Email)를 불변으로 설계하면 무결성 보장에 유리합니다.

## 보조 개념 정리
- 값 객체(Value Object): 식별자보다 값 자체가 의미를 갖는 객체
- 방어적 복사: 외부 수정 영향을 막기 위해 복사본을 반환하는 기법
- 사이드 이펙트: 함수/메서드가 외부 상태를 변경하는 효과
