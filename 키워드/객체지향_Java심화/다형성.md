# 다형성

## 정의
다형성은 동일한 메시지(메서드 호출)에 대해 객체 타입에 따라 다른 동작을 수행하는 객체지향 핵심 개념입니다.
상위 타입으로 코드를 작성하고 실제 동작은 하위 구현체가 결정됩니다.

## 특징
- 인터페이스 기반 설계와 함께 쓰면 확장성이 좋아집니다.
- 런타임 바인딩으로 구현 교체 비용을 줄일 수 있습니다.
- 조건문 분기를 줄이고 전략 패턴 같은 구조를 쉽게 적용할 수 있습니다.

## 실제 사용 예시(사례)
- `DiscountPolicy` 인터페이스를 두고 정액/정률 할인 정책을 구현해 상황별로 바꿔 끼웁니다.
- Spring DI로 구현체를 주입받아 운영 환경에서 정책을 교체합니다.

## 보조 개념 정리
- 업캐스팅: 하위 타입을 상위 타입 참조로 다루는 것
- 동적 바인딩: 실행 시점에 실제 호출 메서드가 결정되는 메커니즘
- 전략 패턴: 알고리즘을 캡슐화해 교체 가능하게 만드는 패턴
