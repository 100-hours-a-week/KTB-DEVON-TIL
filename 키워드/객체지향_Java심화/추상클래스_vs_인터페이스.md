# 추상클래스 vs 인터페이스

## 정의
추상클래스는 공통 상태/기본 구현을 공유하기 위한 상속 기반 추상 타입이고, 인터페이스는 행위 계약을 정의하는 타입입니다.
"무엇을 할 수 있는가"를 표현할 때는 인터페이스, "무엇을 공통으로 갖는가"를 표현할 때는 추상클래스가 적합합니다.

## 특징
- 추상클래스는 필드/생성자/구현 메서드를 가질 수 있습니다.
- 인터페이스는 다중 구현이 가능해 유연한 조합에 유리합니다.
- 자바 8+ 인터페이스는 `default` 메서드로 일부 공통 구현도 제공합니다.
- 실무에서는 계약은 인터페이스로, 재사용은 조합/위임으로 해결하는 편이 안전합니다.

## 실제 사용 예시(사례)
- `MessageSender` 인터페이스로 이메일/슬랙 구현을 분리합니다.
- 공통 템플릿 로직이 많으면 `AbstractBatchJob` 추상클래스로 뼈대를 제공합니다.

## 보조 개념 정리
- IS-A: 상속 관계로 타입을 표현하는 방식
- HAS-A: 조합(컴포지션)으로 기능을 포함하는 방식
- 템플릿 메서드 패턴: 공통 흐름은 상위 클래스, 세부 구현은 하위 클래스가 담당
