# 전략 패턴

## 정의
전략 패턴은 알고리즘(행위)을 캡슐화하고 런타임에 교체 가능하게 만드는 패턴입니다.
조건문 분기를 객체 조합으로 대체해 확장성을 높입니다.

## 특징
- 클라이언트는 추상화에 의존해 구현 변경 영향을 줄일 수 있습니다.
- 새로운 전략 추가가 기존 코드 수정보다 확장으로 해결됩니다.
- 전략 객체가 과도하게 많아지면 관리 비용이 증가할 수 있습니다.

## 실제 사용 예시(사례)
- 결제 수단별 처리 로직을 `PaymentStrategy` 구현체로 분리합니다.
- 할인 정책을 `DiscountStrategy`로 분리해 이벤트별 정책을 교체합니다.

## 보조 개념 정리
- 컨텍스트(Context): 전략을 사용하는 주체
- 정책 기반 설계: 변하는 규칙을 분리해 관리하는 방식
- OCP: 기존 코드 변경을 최소화하며 확장하는 원칙
