# 트랜잭션 전파(Propagation)

## 정의
트랜잭션 전파는 트랜잭션이 이미 존재할 때, 내부 메서드 호출이 기존 트랜잭션에 참여할지 새로 시작할지를 결정하는 정책입니다.
Spring `@Transactional`의 `propagation` 옵션으로 제어합니다.

## 특징
- `REQUIRED`: 기존 트랜잭션 참여(기본값), 없으면 새로 시작
- `REQUIRES_NEW`: 항상 새 트랜잭션 시작, 기존은 일시 중단
- `NESTED`: savepoint 기반 중첩 트랜잭션(지원 환경 제한)
- 전파 정책 선택에 따라 롤백 범위와 장애 전파 방식이 달라집니다.

## 실제 사용 예시(사례)
- 주문 처리 본 트랜잭션은 `REQUIRED`로 유지합니다.
- 실패해도 남겨야 하는 감사 로그는 `REQUIRES_NEW`로 분리해 커밋합니다.

## 보조 개념 정리
- Savepoint: 트랜잭션 내부 롤백 지점
- 롤백 전파: 예외 발생 시 상위 호출로 실패가 전달되는 흐름
- 트랜잭션 경계: begin/commit/rollback이 적용되는 범위
