# 트랜잭션 동기화

## 정의
트랜잭션 동기화는 트랜잭션 경계 안에서 DB 커넥션, 세션, 후처리 콜백 등을 일관되게 묶어 관리하는 메커니즘입니다.
Spring은 `TransactionSynchronizationManager`를 통해 스레드 단위로 이를 관리합니다.

## 특징
- 같은 트랜잭션 안에서 동일 커넥션/세션 재사용을 보장합니다.
- 커밋 전/후, 롤백 시점에 콜백을 등록할 수 있습니다.
- 비동기 경계로 넘어가면 스레드 로컬 기반 문맥이 유지되지 않습니다.

## 실제 사용 예시(사례)
- 주문 트랜잭션 커밋 후에만 알림 이벤트를 발행하도록 afterCommit 훅을 사용합니다.
- 트랜잭션 내에서 여러 DAO 호출이 동일 커넥션을 공유해 일관성을 유지합니다.

## 보조 개념 정리
- ThreadLocal: 스레드별 독립 저장소
- afterCommit: 커밋 성공 이후 실행되는 동기화 콜백
- 트랜잭션 경계 누수: 의도한 범위를 벗어난 자원 사용 문제
