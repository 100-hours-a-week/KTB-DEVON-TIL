# @ControllerAdvice 예외 처리

## 정의
`@ControllerAdvice`는 컨트롤러 전역에서 발생하는 예외를 공통 방식으로 처리해 일관된 에러 응답을 제공하는 메커니즘입니다.
에러 포맷 표준화를 통해 클라이언트 처리 복잡도를 줄입니다.

## 특징
- `@ExceptionHandler`로 예외 타입별 처리 로직을 분리할 수 있습니다.
- 상태 코드와 에러 코드를 중앙에서 통제할 수 있습니다.
- 컨트롤러마다 중복되는 try-catch를 제거할 수 있습니다.

## 실제 사용 예시(사례)
- `BusinessException`은 400 계열, `AccessDeniedException`은 403으로 매핑합니다.
- API 응답을 `timestamp`, `code`, `message`, `path` 형태로 통일합니다.

## 보조 개념 정리
- 에러 계약: 클라이언트와 합의된 오류 응답 스키마
- 도메인 예외: 비즈니스 규칙 위반을 나타내는 예외
- 시스템 예외: 인프라/예상치 못한 런타임 오류
