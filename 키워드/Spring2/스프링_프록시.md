# 스프링 프록시

## 정의
스프링 프록시는 대상 객체 앞에 대리 객체를 두어 부가 기능(트랜잭션, 로깅, 보안)을 공통 적용하는 메커니즘입니다.
AOP와 선언형 트랜잭션이 대표적인 활용 사례입니다.

## 특징
- JDK 동적 프록시(인터페이스 기반)와 CGLIB(클래스 기반) 방식을 사용합니다.
- 핵심 비즈니스 코드를 수정하지 않고 공통 관심사를 분리할 수 있습니다.
- 같은 클래스 내부 자기 호출(self-invocation)에는 프록시가 적용되지 않는 점을 주의해야 합니다.

## 실제 사용 예시(사례)
- `@Transactional` 메서드 호출 시 프록시가 트랜잭션 시작/커밋/롤백을 처리합니다.
- `@Around` 어드바이스로 실행 시간 로깅을 공통 적용합니다.

## 보조 개념 정리
- 부가 기능(Cross-cutting Concern): 여러 모듈에 공통으로 필요한 기능
- 위빙(Weaving): 대상 코드에 부가 기능을 결합하는 과정
- Self-invocation: 객체 내부에서 자기 메서드를 직접 호출하는 패턴
