# 인증 vs 인가

## 정의
인증(Authentication)은 "누구인가"를 확인하는 과정이고, 인가(Authorization)는 "무엇을 할 수 있는가"를 판단하는 과정입니다.
서비스 보안은 인증 이후 인가 규칙으로 리소스 접근을 제어해야 완성됩니다.

## 특징
- 인증 수단: 비밀번호, OTP, OAuth, JWT 등
- 인가 기준: 역할(Role), 권한(Permission), 정책(Policy)
- 인증만 있고 인가가 약하면 권한 상승 취약점이 발생할 수 있습니다.
- API 설계에서 엔드포인트별 권한 매핑이 명확해야 합니다.

## 실제 사용 예시(사례)
- 로그인 시 JWT 발급으로 인증을 완료합니다.
- 관리자 API는 `ROLE_ADMIN` 사용자만 접근 가능하게 제한합니다.

## 보조 개념 정리
- RBAC: 역할 기반 접근 제어
- ABAC: 속성 기반 접근 제어
- 최소 권한 원칙: 업무 수행에 필요한 최소 권한만 부여하는 원칙
