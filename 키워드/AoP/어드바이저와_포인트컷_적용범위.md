# 어드바이저와 포인트컷 적용 범위

## 정의
어드바이저는 포인트컷과 어드바이스를 묶어 "어디에 어떤 부가 기능을 적용할지" 정의한 단위입니다.
적용 범위 설계가 잘못되면 성능 저하나 오작동이 발생할 수 있습니다.

## 특징
- 포인트컷 표현식이 넓을수록 적용 대상이 많아집니다.
- 어드바이스 순서가 다르면 동작 결과가 달라질 수 있습니다.
- 패키지/애노테이션 기반 범위 제한으로 안전성을 높일 수 있습니다.

## 실제 사용 예시(사례)
- `..service..` 패키지에만 성능 로깅 어드바이저를 적용합니다.
- 트랜잭션 어드바이저보다 로깅 어드바이저를 바깥에 두어 전체 실행 시간을 측정합니다.

## 보조 개념 정리
- Advisor: Pointcut + Advice 조합
- Ordered: 어드바이스 실행 순서 제어 인터페이스
- 적용 범위 과다: 불필요한 프록시/로깅으로 오버헤드가 커지는 문제
