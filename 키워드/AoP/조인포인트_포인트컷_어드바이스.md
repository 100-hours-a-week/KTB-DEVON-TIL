# 조인포인트, 포인트컷, 어드바이스

## 정의
AOP에서 조인포인트는 부가 기능을 적용할 수 있는 실행 지점이고, 포인트컷은 그중 실제 적용 대상을 고르는 표현식입니다.
어드바이스는 선택된 지점에서 실행되는 실제 부가 로직입니다.

## 특징
- 조인포인트: 메서드 실행 같은 후보 지점
- 포인트컷: `execution(...)` 등 조건으로 적용 범위를 제한
- 어드바이스: `@Before`, `@After`, `@Around` 등 시점별 동작 정의
- 범위를 넓게 잡으면 의도치 않은 성능 저하가 생길 수 있습니다.

## 실제 사용 예시(사례)
- `com.example.service..*` 패키지 메서드만 로깅 대상으로 포인트컷을 지정합니다.
- `@Around` 어드바이스에서 처리 시간 측정 후 슬로우 로그를 남깁니다.

## 보조 개념 정리
- Aspect: 포인트컷 + 어드바이스를 묶은 모듈
- 위빙: 부가 기능을 대상 코드에 연결하는 과정
- Advice Ordering: 여러 어드바이스 실행 순서 제어
